1. What this game is

Echoes of the Hearth — web-based 2D isometric co-op survival sandbox RPG. Players ("Keepers") wake on the Whispering Woods island of a dying archipelago world ("The Hearth") infested by a cosmic parasite ("The Blight"). To win: climb the tech tree, sail between five far-apart islands, awaken all 4 Titan Monoliths (one per biome island), build the World Engine on the central Core Void island, and survive a 4-minute final assault.

Full design doc: PLAN.md. The design pillars: web-first (Phaser 3 + TS + Vite), zero-budget assets (hand-written SVGs + procedural textures + procedural audio), reactive simulation (ecosystem decay, weather, infection), co-op required difficulty.

2. Quickstart & workflows

npm install
npm run server       # authoritative WebSocket game server on :8081
npm run server:dev   # same, but F9 in-game grants a full dev kit (cheats)
npm run dev          # Vite dev server on :5173 (client) — open several tabs for co-op
npm run build        # tsc --noEmit type-check THEN vite build  (both must pass)
node test.mjs        # full protocol test suite (server must be running)
Definition of done for ANY change: npm run build passes (type-check + bundle), then start a fresh server and node test.mjs passes all stages. If you changed server/shared code, extend test.mjs with a stage covering it.
  • World/persistent state lives in server/save.json (auto-saved every 30s). Delete it for a fresh world — stale saves from an older seed/format are skipped by seed check but a format change needs manual deletion.
  • After changing client code, the browser needs a hard refresh (Ctrl+Shift+R).
  • Kill stale node processes before starting the server — port 8081 conflicts have wasted debugging time before: Get-Process node | Stop-Process -Force (PowerShell).

4. Architecture & core principles

Browser A ──ws──┐
Browser B ──ws──┤──► server/index.js (authoritative: inventory, combat, building,
Browser C ──ws──┘        gathering, digging, weather, creatures, persistence)
        ▲
        │  both sides independently run genWorld(seed) from shared/world.js
        ▼
   src/main.ts (rendering, input, movement prediction, visuals-only state)
  1. Deterministic shared worldgen. The server sends only a seed; client and server both call genWorld(seed) and get identical tiles/elevation/nodes/veins/bergs. Only mutations (chopped nodes, mud, structures, digs…) are synced. Never send the map.
  2. Server is authoritative for everything that matters: inventory, crafting costs, gathering yields, damage, structure HP, digging. It validates distance, tools, stations, and rate-limits (250–400ms) every action. The one exception: movement is client-authoritative (client sends positions; server trusts them). Keep new mechanics server-validated.
  3. Tile index convention: a tile is one integer i = y * SIZE + x (SIZE=320). Decode: x = i % SIZE; y = (i / SIZE) | 0. All protocol messages use tile indices.
  4. Isometric math (TW=64, TH=32): screen sx = (x−y)*32 + offX, sy = (x+y)*16, where offX = (SIZE−1)*32. Inverse (used for mouse picking and chunk bounds): x = (sy/16 + (sx−offX)/32)/2, y = (sy/16 − (sx−offX)/32)/2, floor both. A tile's top diamond vertex is at (sx, sy); its center is (sx, sy+16). Depth-sorting: every world object sets depth = its screen y.
  5. Zero-asset pipeline: art = small hand-written SVGs in assets/; character parts, weather particles, light glows = runtime-generated textures; audio = Web Audio synthesis. When adding art, follow the existing flat-shaded SVG style with a drop-shadow ellipse.

5. Shared modules

shared/world.js — genWorld(seed) returns:

FieldTypeMeaning
tilesUint8ArrayBiome per tile. T = {GRASS:0, SAND:1, SNOW:2, MUD:3, WATER:4, BLIGHT:5}
elevUint8Array0 water, 1 plain, 2 hill, 3 peak. Rendered 10px higher per level; moves >1 level are cliffs (2 while jumping).
veinsUint8ArrayUnderground minerals: 0 none, 1 iron, 2 diamond. Only meaningful under diggable tiles.
nodesMap<i,kind>Harvestables. NODE = {TREE:0, BOULDER:1, BUSH:2, STONE:3, CRYSTAL:4, STARMETAL:5}
bergsSet<i>Iceberg water tiles ringing the Frozen Spire (radius < 72 of isle center).

World geography (SIZE=320): five islands in open ocean, gaps ≈ 80 tiles (> 2 screens — islands are invisible from each other). ISLES = [[80,80] Woods/GRASS, [240,80] Dunes/SAND, [80,240] Spire/SNOW, [240,240] Marsh/MUD], radius ≈ 56 (ISLE_R) with coast noise; bergs ring the Spire to radius 95. Elevation uses gentle thresholds plus a ridged-noise pass that carves mountain chains — expect heavy relief (~⅓ of land at each level). MONOLITHS === ISLES (index order matters — client's MONO_NAMES matches it). CORE=[160,160], Core Void island radius 14. Helpers: findSpawn(world) (starts near 68,68 on Woods), nearestLand(world,x,y), DIGGABLE(world,i) (grass/sand/snow only — never Marsh, water, blight).

Changing anything inside genWorld changes the world for everyone and invalidates server/save.json semantics (structures may end up on water). If you change worldgen, bump the default seed or tell the user to delete the save.

shared/defs.js

RECIPES (cost / station / flags: tool, gear, place), NODE_DEF (hp, required tool, yield, respawn), STRUCT_HP, WOODEN (erodes in night Blight Storms unless near campfire), RESOURCES (inventory bar order), PLACEABLES, NAMES (display names), emptyInv(), canAfford / pay. Add every new item to: RECIPES + NAMES + emptyInv() + (if resource) RESOURCES + ui.ts icon map.

6. Network protocol reference (JSON over ws, port 8081)

Client → Server

atk{}Melee. Rejected when z=1. Hits nearest creature/animal ≤2.4, else demolishes nearest structure (double dmg, refunds half cost). Dmg: isword 5 / sword 3 / axe 2 / else 1 (uses equipped, not owned).

usecore{i}Awaken monolith index 0-3 (needs core item, within 3 tiles).
water{}Collect water when adjacent to a water tile (max 10).
use{k}Consume: 'water' (+4 thirst) or 'cookedmeat' (+5 hunger).
anim{a}Cosmetic relay: 'axe'|'pick'|'sword'|'isword'|null (swing) or 'j' (jump).
dev{}Grants full kit if server started with DEV=1, else polite refusal.

7. Server internals (server/index.js)

State maps (all keyed by tile index unless noted): nodeHp (damaged nodes), removed (i → respawn timestamp), mudTiles, sectorChops (16×16 sector counters), structures (i → {kind,hp,owner,dir,lvl}), digs, torches, furn (i → {kind,owner}), brokenBergs, infected (i → cure timestamp), players (sessionId → player), creatures, animals, mono[4], wave, profiles (token → saved player).

The 200ms tick, in order: day/night clock (300s/day) → weather roll (~90s avg, 40-80s duration) → infection decay (2min) → monster spawn (cap scales with monolith count & night; spawns at Core, at infected tiles, or near players at night) → monster AI (wisp drifts+infects; brute prefers structures, 3 dmg; others chase nearest surface player — scent radius 40 if carrying essence/meat, else 26; structures in the path get attacked; contact damage) → wildlife spawn (player-biased, per-island species) + wander/flee → every 5s: hunger/thirst drain (~10/7min), env damage (sandstorm→shelter by structure; snowstorm→campfire; desert day heat→Heat Cloak; Spire cold→Fur Cloak; skipped underground), campfire regen → every 10s at night: Blight Storm erosion of WOODEN structures not near a campfire → wave timer → broadcast cre.

Persistence: loadSave()/saveGame() ↔ server/save.json, every 30s + SIGINT. Saves world mutations + day/time/mono/won + profiles. Player profiles restore on hello by token. This is THE seam for swapping in a real DB later.

8. Client internals (src/main.ts — scene class "Hearth")

  • Terrain = streamed chunks. The 320×320 map is far too big for one texture. ensureChunks() (called every frame) creates 1024×512 RenderTextures covering the camera (+margin) and evicts beyond 24. drawChunk inverse-projects the chunk rect to a tile range and paints tiles bottom-up with elevation: each tile is drawn elev times stacked 10px apart (skirts become cliff faces). Tile mutations go through setTileMut(i, key|null) which repaints the tile + its 3 southern neighbors on all loaded chunks.
  • Three Z layers. z=0 surface, z=1 underground, z=2 shelter interior. setZ() swaps visibility: surface fades to alpha 0.15, layer sprites show (cave: ugFloor/ugRock/ugOre/torches; interior: generated wood floor + furnSpr), creatures/animals hide, other players show only on the same layer. Non-surface positions use flat iso() + 16 (no elevation). Interior room size = Chebyshev radius = shelter lvl (3×3 → 7×7), anchored at shelterAnchor; enter/exit via E with the shared 900ms zToggleAt cooldown. All server z-guards use q.z !== 0 (attack rejection, monster targeting/contact, weather immunity) so both non-surface layers are protected — keep that pattern for any new layer. addDug(i) maintains floor sprites + rock-wall sprites around tunnels + ore glints from veins. Darkness: darkRT screen-overlay filled black 0.93, with radial-gradient light holes erased at the player (glow-s), torches and mineshafts (glow-l).
  • Movement is screen-relative WASD mapped onto iso axes, ~4.4 tiles/s (2.2 on mud, 6.2 sailing). blockedAt() is the single collision gate: bounds → (z=1: only dug tiles) → (z=2: stay inside the shelter room) → water (passable only with a boat) → cliffs (ascent-only: climbing >1 level blocked, >2 while jumping; dropping any height is allowed and the SERVER applies fall damage) → structures (shelters are enterable/walkable).
  • Boats: walking onto water with a boat item auto-boards (sboat preferred); walking ashore disembarks; boat sprite follows under the rig. Server destroys wooden boats near icebergs (see pos msg).
  • Rig (rig.ts): characters/tools are generated textures; walk cycle & swings are code (tweens). Textures: p-* body parts, i-axe/i-pick/i-spick/i-sword/i-isword hand items. Every equippable tool needs an i-<key> texture or Phaser shows a green missing-texture box.
  • UI (ui.ts) is DOM. It rebuilds innerHTML only when a JSON signature changes and uses delegated click listeners attached once. Never rebuild DOM per-frame (breaks clicks — see pitfalls). Layout: minimal on-screen #quickbar (hotkey tools + water/food), full inventory in the #invModal "Bag" (key I), controls/instructions in the static #helpModal (key H, content lives in index.html), crafting panel (key C). ESC closes modals and cancels placement. Selecting a placeable from the Bag auto-closes it; when z=2 the placement click sends furn instead of build.
  • Occlusion fade: any structure in front of (deeper than) the player within ~64px fades to 0.35 alpha.
  • Audio (audio.ts) is entirely synthesized; unlocks on first user gesture. Weather beds crossfade; footsteps keyed by surface; growls scale with nearest-monster distance; generative pad/pluck score, darker at night.
  • Clue notes: NOTE_DEFS in main.ts — one per island near its monolith, chained hints (Woods→Dunes→Marsh→Spire→Core). Client-only lore; E to read.

Survival numbers

HP 10. Hunger empties in ~10min, thirst ~7min; at 0 → 1 dmg/5s. Water: E at a lake edge, click 💧 to drink (+4). Cooked meat +5. Campfire: regen 1hp/5s within 4 tiles, blocks Blight-Storm erosion within 6, respawn point (nearest own) — but an owned Bed (furniture) outranks campfires. 10. How-to: common change recipes

Add a craftable item

  1. shared/defs.js: RECIPES + NAMES + emptyInv() (+ STRUCT_HP/PLACEABLES if placeable, + RESOURCES if a raw resource).
  2. src/ui.ts: icon map entry (+ add to the inventory-bar display list if it's an item).
  3. If it's a tool: add to HOTBAR arrays in both ui.ts and main.ts, generate an i-<key> texture in rig.ts, add damage case in server atk.
  4. If placeable: sprite SVG + STRUCT_SPR size entry in main.ts (preload loops pick it up automatically).

Add a resource node

  1. defs.js: NODE, NODE_KEYS, NODE_DEF.
  2. world.js: placement rule in genWorld.
  3. SVG in assets/sprites + NODE_SPR entry in main.ts.
  4. Nothing server-side — gather flow is generic.

Add a monster

  1. server: CRE_TYPES entry + spawn condition in tick + any special AI branch.
  2. client: SVG + preload + CRE_TEX entry (+ special tween if wanted).

Add a network message

  1. Handler in server message switch (validate: distance, rate-limit, inventory).
  2. Client: send where needed; add case to onMsg.
  3. If it mutates persistent state, add to saveGame()/loadSave().
  4. Add a test stage in test.mjs.

11. Testing & dev mode

test.mjs spins up two real WebSocket clients against a running server and walks the full game: join/sync, movement relay, gather→craft→build, validation rejections, water/food, demolition+refund, wildlife sim, mineshaft chamber, underground digging, underground attack guard, torch placement, iceberg collision. It recomputes the world from the seed to find real node positions. Patterns to reuse: c.wait(type) polls for a message type; filter stale periodic messages (stat, cre, dig) from c.msgs before waiting for a fresh one — a past test bug.

Dev mode: npm run server:dev then F9 in the browser = full kit (all tools/gear/materials). Refused unless the DEV env var is set. Persistence test pattern (proven): join with a fixed token, mutate, wait for the 30s autosave, kill server, restart, rejoin same token, assert restored inventory.

12. Known pitfalls — every one of these caused a real bug

Vite/esbuild does NOT type-check. A reference to an undeclared class field (this.monoSpr) built fine and crashed at runtime, freezing the whole game loop (uncaught exception in update() every frame). That's why npm run build now runs tsc --noEmit first. Never remove that step. If the game "freezes", check the browser console — it's almost always an exception thrown per-frame from update().
Never rebuild DOM innerHTML every frame. The crafting panel originally regenerated its buttons 60×/s, so mousedown/mouseup landed on different elements and clicks never fired ("crafting is broken"). Rule: rebuild only when a state signature changes; attach listeners once via delegation (see ui.ts).
Don't shadow Phaser Scene properties. A scene field named time collided with Phaser's Scene.time Clock (renamed to wtime); Rig.body collided with the physics property (renamed torso). Check the Phaser Scene/GameObject API before naming fields.
PowerShell mangles UTF-8. Piping file content through Get-Content | Set-Content corrupted em-dashes and emoji (mojibake in server messages). Use the Edit tool / [IO.File]::ReadAllText/WriteAllText for files containing non-ASCII.
Stale processes & caches: an old server holding :8081 made new code appear broken; an old browser tab ran outdated client code. Always kill node processes and hard-refresh after changes.
Symmetric sprites make flips invisible. Wall "rotation" (flipX) did nothing because the original wall SVG was horizontally symmetric. Isometric direction variants must be drawn along a tile diagonal (see current wall.svg: parallelogram at 2:1 slope, ±20px span so neighbors overlap ~8px).
Hold-vs-press input: E is polled every frame (hold-to-gather). The mine enter/exit toggle flip-flopped every 300ms while E was held until a 900ms zToggleAt cooldown was added. Any state toggle triggered from polled keys needs its own cooldown.
Ordering matters in interact(): current priority is (z=2: chest → exit) → (z=1: shaft exit → dig) → mine entrance → shelter enter → notes → nodes → monolith → water. The mine/shelter checks must precede nodes or nearby trees steal the keypress forever; all layer toggles share the 900ms zToggleAt cooldown.
Phaser missing texture = black/green box. Equipping the stone pickaxe showed one because i-spick was never generated. Every new equippable/tool/creature type needs its texture registered before use.
One giant RenderTexture will blow up GPU memory. The map is rendered via streamed 1024×512 chunks for a reason (a full-map texture would be ~12288×10280 ≈ 500MB). Keep using ensureChunks/drawChunk/setTileMut; don't "simplify" back to one texture.
